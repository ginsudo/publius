# Federalist corpus

This directory holds the structured Federalist Papers corpus for Publius.

## Files

- `raw/pg1404.txt` — verbatim text from Project Gutenberg eBook #1404, fetched 2026-04-25. Untouched.
- `raw/pg1404.sha256` — SHA-256 of the raw file, for drift detection on future re-fetch.
- `parse.ts` — one-shot parser. Reads the raw file, emits `federalist.json`. Idempotent. Re-runnable.
- `federalist.json` — structured corpus, all 85 papers. Conforms to the cross-corpus base schema in [`data/SCHEMA.md`](../SCHEMA.md); Federalist-specific extension is documented in `schema.md`.
- `schema.md` — Federalist-specific schema notes (extension fields and editorial calls). For the universal base, see [`data/SCHEMA.md`](../SCHEMA.md).
- `data_quality_issues.md` — anomalies found during parsing, surfaced for editorial review. Regenerated each parser run.
- `README.md` — this file.

## Provenance

- **Source:** Project Gutenberg eBook #1404, *The Federalist Papers*, by Alexander Hamilton, John Jay, and James Madison. Release date July 1, 1998; updated October 29, 2024. URL: <https://www.gutenberg.org/ebooks/1404>.
- **Editorial lineage:** Per Project Gutenberg's credits, transcribed by The Constitution Society, Anonymous Volunteers, and David Widger. The PG text most closely follows the standard text of the McLean's bound edition (May 28, 1788) numbering and ordering.
- **SHA-256:** `279da3df2bd8b45cea7f435f861e53b93190966066445ecf9484e3c807474670` (recorded in `raw/pg1404.sha256` and embedded in `federalist.json`).

## How to regenerate

```bash
cd data/federalist
node --experimental-strip-types parse.ts
```

The parser is dependency-free: standard library only (`node:fs`, `node:crypto`, `node:url`, `node:path`).

## Verification done in Phase 0

**Counts and structural integrity (automated, in parser):**
- Exactly 85 papers parsed (`FEDERALIST No. N` headings, N = 1..85, no gaps or duplicates).
- Every paper has a parsed dateline, a byline, and a non-empty body.
- Authorship distribution matches the canonical scholarly count: 51 Hamilton, 14 Madison, 5 Jay, 3 joint, 12 disputed (Madison by modern consensus). Total: 85.

**Spot-check (manual, against canonical incipits):**
- Federalist 1 — opens "AFTER an unequivocal experience of the inefficacy of the subsisting federal government..." ✓
- Federalist 10 — opens "AMONG the numerous advantages promised by a well constructed Union..." ✓ (PG omits the hyphen in "well-constructed"; verbatim from source)
- Federalist 51 — opens "TO WHAT expedient, then, shall we finally resort..." ✓
- Federalist 70 — opens "THERE is an idea, which is not without its advocates, that a vigorous Executive is inconsistent with the genius of republican government..." ✓
- Federalist 78 — opens "WE PROCEED now to an examination of the judiciary department of the proposed government." ✓

This is the spot-check method approved for Phase 0. A systematic line-by-line comparison against an independent edition (Library of Congress / Avalon Project / McLean facsimile) is deferred. For the editorial pipeline that produces `plain_english`, a stricter source check should be revisited.

## Data quality — corrections and open issues

See `data_quality_issues.md` (regenerated by the parser on every run).

The editorial standard is: do not silently overwrite the source. Any deviation between `federalist.json` and PG #1404 must either be applied as a logged correction with cited external sources, or left as-is and surfaced for editorial review. Corrections are encoded in the `CORRECTIONS` map in `parse.ts` so they are auditable in one place and survive parser re-runs. The parser refuses to apply a correction unless the `from` value matches what it actually parsed — so an upstream fix to PG #1404 will fail the build rather than re-corrupt the data.

Current state: one correction applied (paper 26 dateline 1788 → 1787, verified against Founders Online and three secondary sources); zero open issues.

## Editorial decisions encoded in the parse

These reflect calls made during Phase 0; if they need to change, change the parser and regenerate.

1. **Salutation as first paragraph.** "To the People of the State of New York:" appears as `paragraphs[0]` on every paper. It is part of each paper as originally published; not stripped.
2. **`PUBLIUS` signature stripped.** The closing signature is metadata, not body text. Removed during parse.
3. **Footnotes preserved inline as trailing paragraphs.** Where PG #1404 prints footnotes after the `PUBLIUS` signature (e.g. paper 78), they remain in `paragraphs` at the end. Not split into a dedicated field. To revisit when the footnote presentation standard is set.
4. **Title and dateline disambiguation.** A handful of papers (18, 39, 45, 58) merge the title with the dateline on one line in PG. Parser splits these by locating the dateline regex first; everything before is title, everything after is body.
5. **Disputed twelve attributed to Madison.** Following Mosteller and Wallace (1964). The PG #1404 byline for each of these twelve also reads "MADISON". The dispute is recorded in `authorship_note`, not erased.
6. **Joint papers (18, 19, 20) labeled "Hamilton and Madison" with status `joint`.** PG #1404 byline reads "MADISON, with HAMILTON" for all three; this preserves the joint character without elevating either name above the other.
