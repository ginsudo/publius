# Tocqueville corpus

This directory holds the structured *De la démocratie en Amérique* corpus for Publius. The source of record is the French original (Pagnerre 1848 edition), per the project's standing decision (see `DECISIONS.md`, "Tocqueville: French original as source of record").

## Files

- `raw/tome1.txt` … `raw/tome4.txt` — verbatim text from Project Gutenberg eBooks #30513–#30516 (the four Pagnerre 1848 tomes), fetched 2026-04-25. Untouched.
- `raw/tomes.sha256` — SHA-256s of the raw files, for drift detection on future re-fetch.
- `parse.ts` — one-shot parser. Reads the four raw tomes, emits `tocqueville.json`. Idempotent. Re-runnable.
- `tocqueville.json` — structured corpus, 124 items (2 avertissements + 2 introductions + 93 chapters + 26 end-notes + 1 appendix). Conforms to the cross-corpus base schema in [`data/SCHEMA.md`](../SCHEMA.md); Tocqueville-specific extension is documented in `schema.md`.
- `schema.md` — Tocqueville-specific schema notes (extension fields, kind taxonomy, ID conventions, editorial calls). For the universal base, see [`data/SCHEMA.md`](../SCHEMA.md).
- `data_quality_issues.md` — anomalies found during parsing, surfaced for editorial review. Regenerated each parser run.
- `README.md` — this file.

## Provenance

- **Source:** Project Gutenberg eBooks #30513–#30516, *De la démocratie en Amérique*, by Alexis de Tocqueville. Four physical tomes corresponding to two scholarly volumes:
  - Tome 1 (PG #30513) and Tome 2 (PG #30514) = Volume I (originally published 1835)
  - Tome 3 (PG #30515) and Tome 4 (PG #30516) = Volume II (originally published 1840)
- **Edition:** Pagnerre 1848 — the 12th edition for Vol I and the 5th for Vol II. This edition adds the Vol I "Avertissement de la dixième édition" (a forward Tocqueville composed for the 1848 reprint) and the "Examen comparatif de la Démocratie aux États-Unis et en Suisse" appendix (also 1848). Both are captured as standalone items, dated `1848-01-01`, so a downstream consumer can distinguish original-volume content from Pagnerre additions by date alone.
- **Editorial lineage:** Per Project Gutenberg's credits, transcribed by volunteers from the Bibliothèque nationale de France (BnF/Gallica) copies of the Pagnerre 1848 tomes. URL: <https://www.gutenberg.org/>.
- **SHA-256s:** recorded in `raw/tomes.sha256` and embedded per-tome in `tocqueville.json` under `source.tomes[].sha256`.
- **Edition-fidelity verification (paragraphing, end-note assignment, etc.) against an independent scholarly Pagnerre transcription is deferred to Phase 4** (translation work). See `DECISIONS.md`, "Tocqueville: edition verification deferred to Phase 4."

## How to regenerate

```bash
cd data/tocqueville
node --experimental-strip-types parse.ts
```

The parser is dependency-free: standard library only (`node:fs`, `node:crypto`, `node:url`, `node:path`).

## Verification done in Phase 0

**Counts and structural integrity (automated, in parser):**
- All four tomes parse cleanly with PG START/END boilerplate trimmed and no warnings.
- 124 items total: 2 `avertissement`, 2 `introduction` (Vol I introduction + Vol I Part II preamble), 93 `chapter`, 26 `end_note`, 1 `appendix`.
- Chapter counts per (volume, part) match the canonical distribution: Vol I Part I = 8, Part II = 10; Vol II Part I = 21, Part II = 20, Part III = 26, Part IV = 8. Total: 93.
- 340 inline footnotes across 34 items (Tome 1: 170, Tome 2: 146, Tome 3: 6, Tome 4: 18).
- Item IDs are globally unique. Footnote markers are unique within each item.
- Every inline `[N]` marker (whether in `title` or in `paragraphs`) resolves to a footnote in the same item; every footnote has at least one inline reference.
- Every `[TN-X]` marker in a Vol II tome 4 chapter resolves to a standalone end-note item with the matching ID.

**Spot-check (manual, against canonical incipits):**
- Vol I Introduction — opens "Parmi les objets nouveaux qui, pendant mon séjour aux États-Unis, ont attiré mon attention…" ✓
- Vol I Part I Ch. I — "Configuration extérieure de l'Amérique du Nord." ✓
- Vol I Part II preamble (the unmarked authorial opening to Part II in tome 2) — opens "Jusqu'à présent, j'ai examiné les institutions, j'ai parcouru les lois écrites…" ✓
- Vol II Avertissement — opens with Tocqueville's reference to the five years between the two volumes ("J'ai traité le premier sujet dans l'ouvrage publié par moi il y a cinq ans…") ✓

This is the spot-check method approved for Phase 0. A line-by-line comparison against an independent scholarly Pagnerre transcription is deferred to Phase 4 (see provenance note above).

## Data quality — corrections and open issues

See `data_quality_issues.md` (regenerated by the parser on every run).

The editorial standard is the same as Federalist: do not silently overwrite the source. Any deviation between `tocqueville.json` and the PG tomes (#30513–#30516) must either be applied as a logged correction with cited external sources, or left as-is and surfaced for editorial review. The two parallel mechanisms (`SOURCE_FIXUPS` for raw-text repairs before parsing, `CORRECTIONS` for parsed-field repairs after extraction) follow the Federalist pattern; both have the same precondition discipline (the `from` value must match exactly one occurrence; mismatches throw).

**Phase 0 final state: zero source-text fixups, zero field corrections, zero acknowledged quirks, zero open issues.** The PG transcriptions of the four tomes are clean enough that the parser ran end-to-end with no editorial intervention. The fixup/correction mechanisms remain in place for use if a deviation is discovered later.

## Editorial decisions encoded in the parse

These reflect calls made during Phase 0; if they need to change, change the parser and regenerate. For the full reasoning behind each call, see `DECISIONS.md`, "Tocqueville corpus: source, schema, and editorial calls."

1. **Volume↔tome distinction surfaced.** Both `volume` (work-level: 1 or 2) and `tome` (physical-source: 1, 2, 3, or 4) are stored per item, so downstream code can address either.
2. **Item kinds discriminated by `tocqueville.kind`.** One of `avertissement`, `introduction`, `chapter`, `end_note`, `appendix`. The universal `title` field carries the source-language label.
3. **Pagnerre 1848 augmentations dated 1848.** Vol I's "Avertissement de la dixième édition" and the Vol I appendix carry `date: "1848-01-01"`. Original-volume items carry the original-volume year (1835 or 1840). This makes the original/augmentation distinction queryable by date.
4. **Vol I Part II preamble captured as a standalone `kind: "introduction"` item.** The opening of tome 2 ("Jusqu'à présent, j'ai examiné les institutions…") has no section heading in the source — it sits between the inner header "EN AMÉRIQUE." and the first chapter. Captured at `tocqueville:vol1.preamble.part2` so the prose isn't lost or silently merged into another item.
5. **End-note IDs scoped per-tome.** The Pagnerre 1848 edition resets end-note lettering per tome, so Vol I tome 1's note "A" and Vol I tome 2's note "A" are distinct items. ID format `tocqueville:vol<volume>.t<tome>.notes.<letter>` keeps them globally unique and preserves source provenance in the ID. Vol II tome 4's `TN-` prefix (a PG transcriber convention) is preserved verbatim.
6. **Chapter summaries (Vol I tome 1 only) preserved in `chapter_summary`.** The bullet-style summary block that the Pagnerre 1848 Vol I tome 1 prints between each chapter title and body is captured as a single string; `null` everywhere else.
7. **Markers in chapter titles preserved verbatim in `title`.** Some Vol II tome 3/4 chapter titles carry an inline `[N]` or `[TN-X]` marker as part of the title text. The parser keeps these in the `title` field; the cross-reference invariant scans both `title` and `paragraphs` so the marker → footnote / end-note link is verified regardless of position.
8. **`end_notes_referenced` reserved as `[]` in Phase 0.** The schema reserves an array field on every item for chapter→end-note references, populated editorially in Phase 4. Tocqueville references end-notes in prose ("voyez la note A"), not via a uniform inline marker, so this cannot be reliably auto-derived. Shipping the field shape now keeps the schema stable across Phase 0 → Phase 4.
