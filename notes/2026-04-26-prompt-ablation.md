# Session Notes — 2026-04-26 — Phase 1.2 prompt ablation

Second session today. Built the Phase 1.2 Q&A test harness, locked the v0.1 system prompt and the 15-question test set, ran a v0.1 A/B (full prompt vs. closing section removed), then ran a five-clause ablation against the closing section. Reading the results is deliberately deferred to a separate session.

## What was built

### v0.1 system prompt locked (commit `59b3163`)

- `prompts/system-prompt.md` — v0.1 of the Q&A system prompt. The file mixes a human-only preface and design notes with a model-facing prompt; a structural-convention note near the top defines the boundary.
- `prompts/test-questions.md` — 15-question test set across 5 categories: best-student-tier (Q1–Q3), mediocre-but-not-trivial (Q4–Q6), out-of-corpus (Q7–Q9), genuine disagreement (Q10–Q12), capability-matrix probes (Q13–Q15). Each item has an italics `*What it tests:*` line that the harness strips.
- `DECISIONS.md` — appended "Q&A System Prompt v0.1 (Phase 1.2)" entry recording v0.1 lock and the ablation intent.

### Test harness (commit `d5fc2c8`)

- `prompts/eval/lib.ts` — extraction and formatting helpers, plus a minimal Anthropic HTTP client (no SDK):
  - `extractPrompt(filePath)` — pulls the section between a `## The prompt` heading (with optional version suffix) and the next `---` rule. Everything outside that span is human-only.
  - `parseQuestions(filePath)` — pulls `### Q<N>` blocks, stopping at `*What it tests:*` or the next heading.
  - `formatHit(h)` / `formatHits(hits)` — labeled metadata blocks preserving every field on `Hit` (corpus, item, kind, paragraph_index/marker, paper, title, authors, authorship_status, date, similarity score) plus chunk text.
  - `askClaude(systemPrompt, userMessage)` — POST to `/v1/messages`, model `claude-sonnet-4-6`, max_tokens 4096.
- `prompts/eval/run.ts` — CLI: `--prompt --questions --output --k --limit --questions-filter`. Logs the extracted prompt to stderr at startup as a verification aid. Output markdown carries a header (timestamp, paths, sha256 of the extracted prompt, model, top-K), then per-question (question quoted, hit list with metadata summary, model answer verbatim, token-usage footer).
- Zero new npm dependencies. Imports `loadEnv` from `data/eval/lib.ts` and `queryIndex` from `data/eval/query.ts`. Sequential question processing within a run.
- `--questions-filter Q1,Q3,Q14` flag added during the ablation slice — comma-separated IDs, raises on unknown IDs.

### Ablation prompt variants (commit `d5fc2c8`)

Six variant prompt files alongside `system-prompt.md`, all generated to be character-identical to v0.1 except for the targeted removal:

- `system-prompt-v0.1-no-closing.md` — entire closing "What you are not" section removed (Run B of the A/B).
- `system-prompt-v0.1-no-clause1.md` — verdict-rendering clause removed.
- `system-prompt-v0.1-no-clause2.md` — civics-tutor clause removed.
- `system-prompt-v0.1-no-clause3.md` — interpretive-school-alignment clause removed.
- `system-prompt-v0.1-no-clause4.md` — opinions/corpus-doesn't-have-opinions clause removed.
- `system-prompt-v0.1-no-clause5.md` — corpus-has-arguments-not-opinions tail clause removed.

The clause-level variants were generated by a one-shot script with single-occurrence assertions (refuses to write unless the find-string matches exactly once). All six diffs against `system-prompt.md` were inspected inline before any run kicked off.

## What was tested

### Run A / Run B — full prompt vs. no-closing-section

- `prompts/eval/results-v0.1-runA.md` — full v0.1 against all 15 questions.
- `prompts/eval/results-v0.1-runB.md` — `system-prompt-v0.1-no-closing.md` against all 15 questions.

### Five-clause ablation against the closing section

Only the categories where each clause was hypothesized to matter were tested against each variant. A new fresh baseline of full v0.1 was run alongside, on the relevant question subsets, since Run A's outputs were generated before the harness was finalized:

- `prompts/eval/results-v0.1-baseline-fresh.md` — full v0.1, Q1, Q3, Q5–Q14 (12 questions).
- `prompts/eval/results-v0.1-no-clause1.md` — Q1, Q9, Q14.
- `prompts/eval/results-v0.1-no-clause2.md` — Q6, Q8, Q10.
- `prompts/eval/results-v0.1-no-clause3.md` — Q3, Q5, Q7.
- `prompts/eval/results-v0.1-no-clause4.md` — Q10, Q11, Q13.
- `prompts/eval/results-v0.1-no-clause5.md` — Q6, Q12, Q14.

Total ablation calls: 27 (12 fresh baseline + 15 across the five variants). All 27 plus all 30 A/B calls completed with `end_turn`; no truncations, no API errors. A pre-launch sanity check on the no-clause4 stderr-logged extracted prompt confirmed extraction cut at the right place for the largest structural change.

A one-question harness verification artifact (`prompts/eval/results-verification.md`, Q1 only) is untracked and stays untracked — it served as the harness sanity check before the A/B kickoff.
